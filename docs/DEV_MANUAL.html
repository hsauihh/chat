2. Qt 速成课：10 分钟掌握你需要知道的 Qt

2.1 Qt 是什么？

Qt 是一个 C++ GUI 框架。它提供：

  • Widget（积木）：按钮 QPushButton、文本框 QLineEdit、列表 QListWidget、标签 QLabel 等——把它们组合起来就是界面
  • 信号/槽（电线）：Widget A 发生事件 → 通知 Widget B 做某事。类似于回调函数，但类型安全、支持多对多连接
  • 事件循环（心脏）：QApplication::exec() 启动后，程序不停循环处理鼠标点击、键盘输入、网络数据等事件
  • MOC（代码生成器）：扫描头文件中的 Q_OBJECT 宏，自动生成信号/槽的实现代码

2.2 信号与槽（最重要的一节）

把信号和槽想象成"喊话—应答"：

// 头文件中声明信号和槽
class Teacher : public QObject {
    Q_OBJECT
signals:
    void bellRang();              // 信号：不需要写函数体！MOC 自动生成
};
class Student : public QObject {
    Q_OBJECT
public slots:
    void standUp() { ... }        // 槽：就是一个普通的成员函数
};

// 连接：铃声一响，学生就起立
connect(teacher, &Teacher::bellRang, student, &Student::standUp);

// 触发：铃声响起
emit bellRang();  // → Student::standUp() 自动被调用
信号可以有参数，槽也可以有参数。参数从信号传递到槽。 connect(slider, &QSlider::valueChanged, label, &QLabel::setNum); — 滑块的值变化时自动更新标签文本。

2.3 Q_OBJECT 宏：别忘了它

任何使用了 signals: 或 slots: 的类 必须 在类定义的第一行写 Q_OBJECT。忘记写的话，链接时会报 undefined reference to vtable。

排查方法：看到 vtable 链接错误 → 检查是否写了 Q_OBJECT → 检查头文件是否在 CMakeLists.txt 中被列出 → 重新 cmake 一次。

2.6 .ui 文件：可视化 UI vs 手写代码

本项目的 .ui 文件用 Qt Designer（或 Qt Creator）编辑，保存为 XML 格式。编译时 UIC 编译器 自动把 .ui 编译为 ui_xxx.h，其中生成一个 Ui::XxxWidget 类。

// 在 C++ 中使用 .ui 文件定义好的 UI
Constructor() {
    ui->setupUi(this);         // 必须第一行！创建所有子 Widget
    ui->button->setText("确定"); // 通过 ui-> 访问 .ui 中定义的 Widget
}
.ui 文件 vs 手写代码：复杂布局用 .ui 设计更方便（像 WYSIWYG 编辑器），动态创建 Widget 用代码更方便（如消息气泡）。本项目中两者混用。

2.7 Qt 对象父子树：内存管理

Qt 的对象树自动管理内存。当你 new QWidget(parent) 时，parent 析构时会自动 delete 这个子对象。不需要手动 delete！（除了 ui 指针需要 delete ui。）

5. UI 文件（.ui）的结构与编辑方法

5.1 什么是 .ui 文件

.ui 文件是 XML 格式的 UI 描述文件。你可以用：

  • Qt Creator：打开 .ui 文件 → 可视化拖拽编辑
  • Qt Designer：独立的设计工具
  • 文本编辑器：直接编辑 XML（适合简单改动，如修改按钮文本）

5.2 .ui 文件 ↔ C++ 的对应关系

编译时，UIC (UI Compiler) 将 chatwidget.ui 编译为 ui_chatwidget.h：

// chatwidget.ui 中定义:
<widget class="QPushButton" name="sendBtn">
  <property name="text">
    <string>发送</string>

// 在 C++ 中访问:
ui->sendBtn->setText("发送中...");
connect(ui->sendBtn, &QPushButton::clicked, ...);

5.3 如何修改 .ui 文件

  1. 在 Qt Creator 中打开：双击 .ui 文件 → 进入设计模式
  2. 拖拽 Widget：从左侧工具箱拖拽按钮、文本框等到画布上
  3. 修改属性：在右下角属性面板修改 objectName（代码中的名字）、text、size 等
  4. 设置布局：选中多个 Widget → 右键 → 布局 → 水平/垂直
  5. 保存：Ctrl+S → 重新 cmake → 重新编译

如果不想装 Qt Creator：

直接编辑 XML 也可以。关键在于 name 属性（对应 C++ 中 ui->name）：

<widget class="QPushButton" name="myButton">
  <property name="text">
    <string>我的按钮</string>
  </property>
</widget>

6. 如何添加新功能（5 个实战案例）

添加任何新功能前，先在纸上画出：信号从哪里来 → 经过谁 → 最终到哪里去。这个思维习惯能避免 90% 的 Qt bug。

案例 1：在聊天界面添加表情面板

8. 常见问题排查 (FAQ)

Q: 编译报错 "undefined reference to vtable for XxxWidget"

原因：忘记写 Q_OBJECT 宏，或者头文件没有被 MOC 扫描到。

解决：

  1. 确认类定义中写了 Q_OBJECT
  2. 确认头文件在 CMakeLists.txt 的 HEADER_LIST 中
  3. cd build && cmake .. && make — 只改过头文件时必须要重新 cmake
Q: 信号和槽连上了但槽函数没有被调用

原因：

  • 信号和槽的参数签名不匹配
  • 使用了旧式 SIGNAL()/SLOT() 宏，字符串写错了
  • connect 返回的 QMetaObject::Connection 没有保存——说明是在对象销毁后才连接的

解决：使用新式函数指针语法 &Class::method，编译期就能检查签名。

Q: 中文输入法无法使用

排查步骤：

  1. 确认 QPlainTextEdit（不是 QTextEdit）用作输入框
  2. 确认构造函数中设置了 setAttribute(Qt::WA_InputMethodEnabled, true)
  3. 尝试修改 main.cpp 中的 QT_IM_MODULE：空字符串、"ibus"、"fcitx"、"xim"
  4. 检查 WSL/Linux 系统中是否安装了输入法：fcitx-diagnose 或 ibus version
Q: .ui 文件修改后没有生效

原因：改了 .ui 文件但没有重新运行 cmake，UIC 没有重新生成 ui_xxx.h。

解决：只要改过 .ui 文件，就要 cd build && cmake .. && make。不能只 make。

Q: 消息气泡没有显示

可能原因：

  • QListWidgetItem 没有设置 sizeHint → item->setSizeHint(bubble->sizeHint())
  • QListWidget 的 spacing 太大导致 item 被遮挡
  • MessageBubble 的 UI 布局有问题，导致高度为 0
Q: 如何在不启动客户端的情况下测试 UI

写一个简单的独立 main.cpp，只创建你要测试的 Widget：

int main(int argc, char *argv[]) {
    QApplication app(argc, argv);
    ChatWidget w;
    w.show();
    return app.exec();
}

9. Qt 信号/槽 API 速查

连接方式

// 方式1：新式（推荐）—— 类型安全，编译期检查
connect(sender, &SenderClass::signalName, receiver, &ReceiverClass::slotName);

// 方式2：Lambda（灵活）—— 适合简单逻辑
connect(btn, &QPushButton::clicked, this, [this]() {
    // 直接写逻辑，不用声明槽函数
    doSomething();
});

// 方式3：带参数的 Lambda
connect(slider, &QSlider::valueChanged, this, [this](int value) {
    label->setNum(value);
});

// 方式4：信号→信号（转发）
connect(a, &A::sig1, b, &B::sig2);  // a的sig1触发时自动emit b的sig2
